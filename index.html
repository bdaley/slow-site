About This Website

This website is intentionally designed to demonstrate multiple web performance anti-patterns. It includes render-blocking resources, unoptimized images, excessive HTTP requests, and unused code - all common issues that slow down real websites.

By studying this site's poor performance, developers can learn to identify and fix similar issues in their own projects. Every performance problem here is deliberate and serves as an example of what NOT to do in production websites.

Use browser DevTools to inspect the Network tab, Performance tab, and Lighthouse to see all the performance issues in action!

Performance Problems Featured

🐌

Render-Blocking Resources

Multiple CSS and JavaScript files loaded synchronously in the head, blocking page rendering and delaying First Contentful Paint.

🖼️

Unoptimized Images

Oversized images in wrong formats, scaled by the browser, with loading="eager" and no modern format alternatives like WebP or AVIF.

📦

Unused Code

Entire CSS and JavaScript files loaded but not used, wasting bandwidth and increasing parsing time without providing any value.

🔗

Excessive HTTP Requests

10+ separate CSS and JavaScript files instead of bundling, creating overhead with multiple round trips to the server.

📝

Unminified Resources

All CSS and JavaScript files are verbose and unminified, with extensive comments and whitespace that increase file sizes unnecessarily.

🌐

No CDN

Third-party libraries like jQuery and Bootstrap served locally instead of from CDNs, missing caching benefits and geographic distribution.

Image Gallery

These images are all 3000x2000 pixels (9MB+ combined), saved as PNG instead of JPG, and loaded with loading="eager" even though they're below the fold!
